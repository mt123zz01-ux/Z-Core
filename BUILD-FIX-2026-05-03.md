# Build Fix - 2026-05-03

Fixed compile blockers reported from Windows Gradle build.

## Changes

- `FakePlayer.java`: removed direct references to removed Minecraft 1.21.11 rotation history fields:
  - `prevYaw`
  - `prevPitch`
  - `prevHeadYaw`
  - `prevBodyYaw`
- `FakePlayer.java`: kept current yaw, pitch, headYaw, and bodyYaw copy behavior.
- `Finder.java`: added suppression for raw/unchecked builder warnings caused by the original decompiled-style enum setting builder. This does not change runtime behavior.
- `BedrockVoidEsp.java`: verified tracer camera uses `Render3DEvent` offsets instead of the removed/invalid `Camera#getPos()` call.

## Verification performed in sandbox

The sandbox cannot download Gradle 9.2.1 from `services.gradle.org`, so a full Gradle compile cannot be completed here. Performed static verification instead:

- No `prevYaw`, `prevPitch`, `prevHeadYaw`, or `prevBodyYaw` references remain.
- No `mc.gameRenderer.getCamera().getPos()` reference remains.
- `Render3DEvent` in `meteor-client-1.21.11-63.jar` exposes `offsetX`, `offsetY`, and `offsetZ`, which are used by `BedrockVoidEsp.java`.
- `Finder.java` is still registered in `ZCore.java`.
